# supabase/

## 목적
Supabase 로컬 개발 환경 설정 및 DB 마이그레이션 파일을 관리한다.
PostgreSQL 스키마(7개 테이블 + RLS)와 Supabase CLI 설정을 담는다.

## 구조

```
supabase/
├── config.toml              # Supabase CLI 프로젝트 설정 (포트, auth, realtime 등)
├── seed.sql                 # 개발용 시드 데이터 (현재 비어있음 — Phase 2에 추가)
├── .gitignore               # .branches/, .temp/ 제외
└── migrations/
    └── 20260408000000_initial_schema.sql  # 초기 스키마: 7 tables + 14 policies + 4 indexes
```

## 로컬 스택 실행

Docker가 실행 중이어야 한다.

```bash
# 로컬 Supabase 스택 시작 (최초 실행 시 Docker 이미지 다운로드)
npx supabase start

# 출력된 API URL / anon key / service_role key를 apps/web/.env.local 에 복사
```

## DB 초기화 / 재현

```bash
# 마이그레이션 + 시드 데이터를 처음부터 다시 적용
npx supabase db reset
```

## 마이그레이션 명명 규칙

```
YYYYMMDDHHMMSS_description.sql
예: 20260408000000_initial_schema.sql
```

새 마이그레이션 생성:
```bash
npx supabase migration new <description>
```

## 테이블 목록 (초기 스키마)

| 테이블 | 설명 |
|---|---|
| `teachers` | 교사 프로필 (Supabase Auth 연동) |
| `sessions` | 수업/퀴즈 세션 |
| `questions` | 퀴즈 문항 |
| `responses` | 학생 응답 |
| `ai_insights` | AI 분석 결과 캐시 |
| `class_drafts` | AI 수업 초안 |
| `thumbs_feedback` | 따봉 업/다운 피드백 |
