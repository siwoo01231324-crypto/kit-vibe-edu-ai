# prompts/ — 핵심 프롬프트 보관

## 목적

개발 과정에서 사용한 핵심 프롬프트 원문을 단계별로 보관한다. AI 리포트 작성 시 참조 자료로 활용한다.

## 파일명 컨벤션

```
{단계}_{순번}_{설명}.md
```

### 단계 접두사

| 접두사 | 단계 |
|--------|------|
| `plan` | 기획 |
| `design` | 설계 |
| `dev` | 개발 |
| `test` | 테스트 |
| `docs` | 문서 |

### 예시

- `plan_01_painpoint-analysis.md` — 기획 단계 페인포인트 분석 프롬프트
- `design_01_architecture-prompt.md` — 설계 단계 아키텍처 설계 프롬프트
- `dev_01_auth-middleware.md` — 개발 단계 인증 미들웨어 생성 프롬프트

## 각 프롬프트 파일 메타데이터

```markdown
# {설명}

- **날짜**: YYYY-MM-DD
- **도구**: Claude Code / GitHub Copilot / 기타
- **목적**: 이 프롬프트로 달성하려는 목표

## 프롬프트 원문

\`\`\`
(프롬프트 전문)
\`\`\`

## AI 응답 요약

(핵심 응답 내용 요약)

## 채택 여부 및 수정 사항

- 채택 여부: 채택 / 수정 후 채택 / 기각
- 수정 내용: (있는 경우)
```
