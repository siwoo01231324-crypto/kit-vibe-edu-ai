### 금지 사항

- API Key, 시크릿 키 등 민감 정보를 AI 프롬프트에 포함하지 않는다
- AI가 생성한 코드를 검토 없이 프로덕션에 반영하지 않는다
- 개인정보(실제 사용자 데이터)를 AI에 입력하지 않는다

### 코드 AI 생성 시 검토 프로세스

1. AI가 코드를 생성한다
2. 팀원이 로직, 보안, 성능 관점에서 검토한다
3. 테스트를 작성하여 동작을 검증한다
4. 검토 완료 후 커밋한다

## 3. AI 생성 코드 품질 관리

### 코드 리뷰 체크리스트

- [ ] 비즈니스 로직이 요구사항과 일치하는가
- [ ] 에러 처리가 적절한가
- [ ] 보안 취약점이 없는가 (XSS, SQL Injection, CSRF 등)
- [ ] 불필요한 코드/주석이 없는가
- [ ] 기존 코드 스타일과 일관성이 있는가

### 보안 취약점 검토 항목

- 사용자 입력 검증 (sanitization)
- 인증/인가 로직 검증
- API Key 등 민감 정보 하드코딩 여부
- OWASP Top 10 항목 점검

## 4. AI 협업 사례집

### 커맨드 기반 워크트리 워크플로우

이슈별로 격리된 개발 환경을 자동 생성하고, AI가 작업 흐름 전체를 관리하는 워크플로우를 운영한다.

**워크플로우 흐름:**
```
/si {이슈번호}     이슈 작업 시작 — 워크트리·브랜치·work 폴더 자동 생성, 이슈 assign
    ↓
/ri                세션 재시작 시 AC 달성 현황 출력, 컨텍스트 복원
    ↓
/plan              구현 계획 작성 (ralplan 컨센서스: Planner → Architect → Critic)
    ↓
(구현 작업)
    ↓
/fi                커밋 + PR 생성 — AI가 변경사항 분석 후 커밋 메시지 자동 생성
    ↓
/ci                PR 머지 확인 후 워크트리·로컬 브랜치 정리
```

**사용 가능한 커맨드:**

| 커맨드 | 단축 | 역할 |
|--------|------|------|
| `/start-issue` | `/si` | 이슈 작업 시작 — 워크트리 생성, 브랜치 생성, docs/work 폴더 구성 |
| `/remind-issue` | `/ri` | AC 현황 출력 + 플랜 검증 + 작업 내역 기록 |
| `/plan` | — | 구현 계획 작성 (OMC ralplan 또는 직접 작성) |
| `/finish-issue` | `/fi` | 변경사항 분석 → 커밋 → PR 생성 |
| `/cleanup-issue` | `/ci` | PR 머지 확인 후 워크트리·브랜치 정리 |
| `/drop-issue` | — | 이슈 중도 포기 → 사유 기록 → 정리 |
| `/backlog-issue` | `/bi` | 새 이슈를 Backlog에 생성 |

**핵심 장점:**
- **격리**: 이슈마다 독립된 워크트리에서 작업하므로 브랜치 간 충돌 없음
- **컨텍스트 복원**: `/ri`로 세션 재시작 시에도 AC 현황·플랜·변경사항을 즉시 파악
- **자동화**: 커밋 메시지 생성, PR 생성, GitHub Project Board 상태 이동이 모두 자동
- **품질 보증**: `/plan`에서 ralplan 컨센서스(Planner → Architect → Critic)를 통해 구현 전 다각도 검토
