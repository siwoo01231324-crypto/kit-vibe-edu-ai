## RALPLAN-DR 요약

### 원칙 (Principles)
1. **실시간 기록 우선**: 사후 재구성은 디테일 손실 — 템플릿/구조를 먼저 만들어 매일 기록 가능하게 한다
2. **심사 기준 정렬**: 심사 기준 2번(AI 활용 능력 및 효율성)에 직결 — 구체적 사례와 정량적 데이터 중심
3. **골격 먼저, 내용 점진적 채움**: 6일간 개발과 병행하므로 완전한 구조를 먼저 세우고 매일 채워나간다
4. **제출 양식 준수**: 공모전 첨부 양식(PDF)을 반드시 따른다
5. **전략 문서 기반**: `05_report-doc-strategy.md`의 상세 구성을 반영한다

### 의사결정 동인 (Decision Drivers)
1. **마감 시간 (04/13 오전)**: 모든 문서가 이 시점까지 완성 + PDF 변환
2. **심사 가점 극대화**: AI 리포트(필수) + AI 활용 지침서(가점) 모두 높은 품질
3. **개발 병행성**: 개발 기간(04/06~04/13)과 문서 작성 동시 진행 — 매일 기록 워크플로우 필수

### 실행 옵션 비교

| 구분 | Option A: 골격 선행 + 점진적 채움 | Option B: 마감 직전 일괄 작성 |
|------|-----------------------------------|-------------------------------|
| 설명 | 지금 전체 구조/템플릿 생성, 매일 로그 기록 | 04/12~13에 몰아서 작성 |
| 장점 | 디테일 보존, 프롬프트 원문 즉시 저장, 스크린샷 누락 방지 | 개발에 집중 가능 |
| 단점 | 매일 10~15분 기록 시간 필요 | 디테일 손실 심각, 프롬프트/스크린샷 복원 불가 |
| 리스크 | 낮음 — 빈 골격 방치 가능성(Step 5 daily 기록으로 방어) | **높음** — 마감 못 맞출 가능성 |

**결정: Option A (골격 선행 + 점진적 채움)**

- PDF 파일 커밋 금지 (.gitignore 규칙) — 로컬 생성 후 메일 제출만
- 5MB 초과 스크린샷 커밋 금지

## ADR (Architecture Decision Record)

- **결정**: 골격 선행 + 점진적 채움 방식으로 AI 리포트 문서를 작성한다
- **동인**: 마감 시간 제약, 실시간 기록의 중요성, 개발 병행 필요성
- **검토한 대안**: (A) 골격 선행 + 점진적 채움, (B) 마감 직전 일괄 작성
- **선택 사유**: Option B는 프롬프트 원문/스크린샷의 사후 복원 불가, 디테일 손실, 마감 리스크 높음
- **디렉토리명**: 전략 문서(05_report-doc-strategy.md)는 `docs/ai-usage/`를 권장하나, 이슈 AC가 `docs/ai-report/`로 명시되어 있으므로 AC를 따름
- **PRD 범위**: PRD(기획문서)는 이 이슈의 AC에 포함되지 않으므로 별도 이슈로 분리한다. 전략 문서 1-2절에서 PRD를 심사 가점 요소로 권장하므로 후속 이슈 생성을 권장함
- **후속 조치**: 04/12에 최종 리포트 초안 리뷰, 04/13 오전 PDF 변환 및 제출
