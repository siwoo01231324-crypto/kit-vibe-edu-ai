## 구현 계획

### 리서치 종합 핵심 인사이트

#### 0. 본질적 문제 — "보이지 않는 관계의 단절"

15개 페인포인트(채점 부담, 낮은 완강률, 이탈 추적 어려움 등)는 **증상**이다. 본질은 하나:

> **교육 현장에서 서로가 서로를 볼 수 없게 되었다.**

- **교사**: 가르치지만, 그것이 닿았는지 모른다 → "나는 왜 가르치는가" (교직만족도 22.7%)
- **학생**: 막혀있지만, 안전하게 말할 곳이 없다 → "나를 봐주는 사람이 없다" (완강률 13.1%)
- **운영자**: 관리하지만, 숫자 뒤의 진짜를 모른다 → "이 교육이 효과 있는지 모른다"

**우리가 만들 것은 도구가 아니라, 이 세 사람 사이에 끊어진 관계를 회복시키는 AI 생태계다.**

#### 1. 시장 기회

글로벌 AI 교육 시장은 USD 5.88B(2024) → 32.27B(2030), **CAGR 31.2%** 성장 중 (Grand View Research). 그러나 기존 12개 솔루션은 **관계가 아닌 기능**에 집중:

| 기존 접근 (기능 중심) | 우리의 접근 (관계 중심) |
|---------------------|---------------------|
| 학생용 AI 튜터 → 교사 배제 | 학생의 신호를 교사에게 전달하는 다리 |
| 교사용 채점 도구 → 학생과 단절 | 교사의 가르침이 학생 성장으로 증명되는 순환 |
| 운영자 대시보드 → 사후 통계 | 교실에서 일어나는 관계의 맥락을 실시간 전달 |

#### 2. 페인포인트 핵심 (관계 단절 구조)

표면적 문제와 본질의 연결:
- 교사 행정업무 **주당 6시간** (OECD 2배) → 학생과의 관계에 쓸 시간을 빼앗김
- K-MOOC 완강률 **13.1%** → "나를 봐주는 사람이 없다"는 학생의 조용한 포기
- 교사 우울 증상 **63.2%** → 가르침이 닿았는지 확인할 수 없는 고립감

#### 3. 수상 전략 핵심

6개 수상작 분석에서 공통 성공 요인:
1. **명확한 페인포인트 타겟** (6/6) — 막연한 "교육 개선"이 아닌 구체적 문제
2. **AI가 핵심 가치 생성** (5/6) — AI 없으면 서비스 불가능한 구조
3. **즉시 작동하는 MVP** (6/6) — 기능 3개 이하, 완성도 100%
4. **수치 기반 임팩트 증명** (4/6) — "채점 30분→5분" 등

실패 프로젝트 패턴: ChatGPT 단순 래핑, 기능 과다·완성도 낮음, 추상적 문제 정의

#### 4. 추천 기술 스택

| 구성 | 선택 | 이유 |
|------|------|------|
| 프론트엔드 | **Next.js 15 App Router** | SSR+API Routes 통합, Vercel 최적화, 1주 MVP 최적 |
| 백엔드 | **Next.js API Routes + Supabase** | 인증·DB 즉시 사용, 무료 한도 충분 |
| AI API | **OpenAI GPT-4o-mini** (또는 Claude Haiku) | 가성비 최고, 한국어 우수, 스트리밍 지원 |
| 배포 | **Vercel Hobby** | 무료, Next.js 공식 지원, 자동 CI/CD |
| UI | **Tailwind CSS + shadcn/ui** | 빠른 프로토타이핑, 반응형 내장 |

#### 5. 500팀 차별화 핵심 전략 — "도구가 아니라 관계"

대부분의 팀이 만들 것: "교사를 위한 AI 채점 도구", "학생용 AI 챗봇", "학습 분석 대시보드"
**우리가 만들 것: 서로가 서로를 다시 볼 수 있게 해주는 AI 생태계**

1. **관계 회복의 수치화** — "채점 30분→5분"이 아니라 "교사가 학생 이해도를 파악하는 시간 0초" (실시간 AI 번역)
2. **안전한 표현 채널** — 학생이 부담 없이 "여기 모르겠어요"를 AI를 통해 전달 → 교사에게 집계·분석되어 도착
3. **가르침의 증거** — "당신의 수업 이후 학생 이해도가 23% 상승했습니다" → 교사 보람 회복의 실시간 피드백
4. **3자 연결 UX** — 교사·학생·운영자가 같은 교실의 다른 시점을 보는 하나의 생태계
