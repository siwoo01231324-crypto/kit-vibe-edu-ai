## 1. Executive Summary

**프로젝트**: AI 기반 교육 피드백 솔루션 — "교사-학생 사이의 끊어진 피드백 루프를 AI로 복원"

**핵심 발견**:
- 한국 EdTech 시장에서 **"감성적 + 교사중심"** 포지셔닝은 완전히 비어 있다. 모든 경쟁사가 "성적 향상", "AI 효율화" 등 기능적 숫자로 설득하는 반면, 교사의 감정과 관계에 호소하는 브랜드는 부재하다.
- 글로벌 성공 사례(ClassDojo, Duolingo, Kahoot 등)의 공통 패턴은 **"감성적 미션 + 단일 강렬한 컬러 + 바이럴 루프"**이다.
- **네이밍 Top 3**: Seeya(시야), Lumi(루미), Echoo(에쿠) — 현행 MirAI 대비 더 높은 발음성·기억가용성·도메인 확보 가능성.

**브랜딩 핵심 공식**:
```
유일한 "감성적 + 교사중심" 포지션
+ Why(관계 회복) → How(60초 피드백 루프) → What(시간 절약 + AI 인사이트)
+ 3층 메시지 구조 (감성 → 실용 → 증거)
```

### 3.3 브랜딩 갭 3가지

| 갭 | 현황 | 기회 |
|----|------|-----|
| **감성 갭** | 모든 경쟁사가 기능·숫자·성적으로 설득. 교사의 감정에 호소하는 브랜드 부재 | "관계 회복"이라는 감성적 Why를 브랜드 중심에 배치 |
| **교사 공감 갭** | 교사를 "도구 사용자"로만 취급. 교사의 존재적 고민("내 가르침이 닿고 있는가")에 응답하는 브랜드 없음 | 교사를 "관계의 핵심 주체"로 존중하는 브랜드 철학 |
| **피드백 루프 갭** | 학습 데이터 수집은 많으나, "학생 → 교사 → 수업 개선"의 폐쇄 루프 실현 제품 없음 | 60초 피드백 → AI 인사이트 → 다음 수업 반영의 완전한 루프 |

## 4. 네이밍 전략

### 4.1 EdTech 네이밍 트렌드

| 유형 | 예시 | 특징 |
|------|------|------|
| **기술적 설명형** | MasterClass, Coursera, Skillshare | 즉시 이해, SEO 유리, 창의성 부족 |
| **감성적 의미형** | ClassDojo, Outschool, BetterUp | 감성 연결, 기억에 오래 남음 |
| **조합형/포트만토** | Duolingo, Pinterest, Kahoot | 독창적, 의미 레이어 다중 |
| **순수 창조형** | Notion, Canva, Spotify | 트레이드마크 용이, 글로벌 확장 |

**2023-2025 트렌드**: 2음절 이하 선호, .ai/.app TLD 활용 급증, "AI-" 접두어 포화 경고, 교육 따뜻함 + 기술 세련됨 균형 추구.

> 출처: [Smart Branding — 300+ EdTech](https://smartbranding.com/edtech-companies-and-their-domain-name-choices/), [Metabrand — Startup Naming](https://www.metabrand.digital/guides/startup-branding-guide/brand-naming)

### 4.2 네이밍 평가 기준

1. **발음성**: 한국어/영어 모두 첫 번째 읽을 때 정확히 발음 가능 (2-3음절)
2. **기억 가용성**: 한 번 듣고 기억 가능, 리듬감
3. **의미 레이어**: 최소 2가지 이상 의미/연상 (서비스 메타포 + 감성)
4. **감성적 공명**: 교육의 따뜻함 + AI의 세련됨 동시 전달
5. **글로벌 확장성**: 다른 언어에서 부정적 의미 없음
6. **도메인 가용성**: .com 또는 .ai 확보 가능

### 4.4 Top 3 추천

**1위: Seeya (시야)** ★29/30
- **의미**: 한국어 "시야(視野, 시각적 범위)" + 영어 "See ya(친근한 인사)" + "See + AI"
- **핵심 메시지**: 교사의 시야를 넓혀주는 AI — "30명을 한 번에 볼 수 있게"
- **감성**: 친근하고 밝으며, 기술적이지 않음. 교육 현장에서 자연스러운 이름
- **도메인**: seeya.ai, seeya.app 탐색 필요

**2위: Lumi (루미)** ★27/30
- **의미**: Lumen(빛) / Illuminate(밝히다) — 수업의 빛, 통찰의 빛
- **핵심 메시지**: AI가 교실의 보이지 않는 것을 밝혀준다
- **감성**: 따뜻하고 밝은 이미지. 페르시아 시인 루미(Rumi)와 연상 → 지혜, 통찰
- **도메인**: lumi.app (일부 사용 중), lumi.ai 변형 가능

**3위: Echoo (에쿠)** ★26/30
- **의미**: Echo(메아리/반향) + -oo(귀여움 강조)
- **핵심 메시지**: 수업의 메아리 — 학생의 목소리가 교사에게 돌아오는 피드백 루프
- **감성**: 서비스 본질("피드백 루프")과 가장 직접 연결, 브랜드 캐릭터 친화적
- **도메인**: echoo.ai, echoo.app 확보 가능성 높음

**현행 MirAI 평가** ★24/30: 3중 의미 레이어(Mirror+AI+미래)는 강하지만, 유사 이름(Mirai Labs 등)이 다수 존재하고 .com 도메인 경쟁이 치열. 감성보다 기술적으로 느껴질 수 있음.

| **따뜻한 관찰자** | 학생의 작은 신호도 놓치지 않는 세심한 동료 교사 | "수업 종료 5분 후, 67%가 미분을 어려워했어요" |
| **신뢰할 수 있는 조수** | 교사의 판단을 돕되 대신하지 않는 비서 | "판단은 당신 것, 저는 정보만 정확히 드립니다" |
| **학생의 대변자** | 발표 불안 없이 목소리를 전달하는 안전한 채널 | "당신의 질문, 선생님이 보셨어요" |
| **현장 이해자** | 교육 현장의 실제 고통을 알고 있는 AI | 과장스럽지 않고, 현실적이고, 실행 가능한 언어 |

### 5.3 Core Values (5개)

1. **관계 우선 (Relationship First)** — 기술은 수단, 교육의 본질은 사람과 사람의 신뢰
2. **신호 증폭, 대체 아님 (Amplify, Not Replace)** — 교사의 직관과 판단을 대신하지 않음
3. **투명성과 설명 가능성 (Explainability)** — AI 판단의 이유를 명확히 설명
4. **현장 친화성 (Practical Empathy)** — 이상적 교육이 아닌 현실의 교실에서 작동
5. **포용적 설계 (Inclusive by Default)** — 한 명의 학생도 소외되지 않도록

## 6. 프레이밍 비판 대응

### 6.1 핵심 비판 (docs/background/11_solution-rethink-proposal.md)

> "프레이밍(관계 회복) ≠ 기능(데이터 수집·집계)" — 이해도 히트맵은 이해도를 측정하는 것이지 관계를 회복하는 것이 아니다.

3대 비판:
1. Why-How-What 사이의 논리적 단절
2. 학생 자발적 참여 전제의 구조적 결함
3. 교사에게 시간 절약이 아닌 추가 업무

### 6.2 브랜딩 입장: Why → How → What

```
┌───────────────────────────────────────┐
│  WHY (존재 이유)                       │
│  "교사와 학생 사이의 단절된             │
│   피드백 루프를 회복한다"               │
├───────────────────────────────────────┤
│  HOW (방법)                            │
│  "수업 직후 60초 피드백 루프"           │
│  학생: 60초 익명 체크인                 │
│  AI: 실시간 클러스터링 + 인사이트 생성   │
│  교사: 다음 수업 시작 슬라이드 자동 생성  │
├───────────────────────────────────────┤
│  WHAT (결과물)                         │
│  "교사의 시간 절약 + 학생 목소리 반영    │
│   + AI 인사이트"                       │
│  수업 준비 31% 단축 (EEF RCT 2024)     │
└───────────────────────────────────────┘
```

### 6.3 비판별 대응

| 비판 | 기존 | 재정의 |
|------|------|--------|
| 관계 회복과 기능 연결 안 됨 | 이해도 히트맵 → 관계 회복? | 60초 피드백 → AI 신호 읽음 → 다음 수업 변화 → **"내 말이 전달되었다"는 관계 경험 발생** |
| 학생 참여 전제 결함 | 수업 중 버튼 누르기 | 수업 후 QR 1회 스캔으로 60초 완료 (교사 주도) |
| 교사에게 추가 업무 | "리포트를 확인하세요" | **"다음 수업 시작 슬라이드가 자동 생성됩니다"** (할 일 삭제) |

### 6.4 3층 메시지 구조

| 레이어 | 메시지 | 타겟 |
|--------|--------|------|
| **감성 (Why)** | "60초면 서로의 마음이 닿습니다" | 교사·학생 공통 |
| **실용 (How)** | "수업 후 60초 피드백 → AI가 다음 수업을 준비합니다" | 교사 |
| **증거 (What)** | "수업 준비 30분 절약, 피드백 반영률 실시간 확인" | 의사결정자 |

### 7.4 Aha Moment — Before/After

**Before** (AI 없음):
- 교사: 90명 학생, 누가 이해하고 누가 못하는지 모름
- 학생: 손 들고 질문하기 부끄러움. 이해 못해도 버팀
- 주당 20시간 행정 업무 + 채점

**After** (AI 사용):
- 교사: 수업 종료 5분 후 "오늘 67%가 미분 어려워함" 인사이트 확인
- 학생: 익명 질문으로 "손 들기 불안" 제거, 자신의 질문이 반영됨
- 주당 8시간 시간 절감 (AI 채점, 리포트 자동화)

## 9. 톤앤매너 & 비주얼 아이덴티티 가이드

### 9.1 톤 스펙트럼

```
[전문적 ←─────────→ 캐주얼]
                  ● 우리 위치
                   
[차가운 ←─────────→ 따뜻한]
                  ● 우리 위치

위치: 따뜻한 + 약간 전문적
- 교사의 신뢰를 위해 "전문성" 필요

- 거창하지 않고, 겸손하고, 함께하는 톤
```
