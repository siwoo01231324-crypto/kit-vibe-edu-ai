# ch0. Executive Summary

## 문제

한국 교육 현장은 **"피드백 루프의 단절"**이라는 구조적 위기에 놓여 있다. 교사의 행정업무 시간은 OECD 평균의 2배(주당 6시간)이며, 교직 만족도는 22.7%에 불과하다. 학생 10명 중 9명이 온라인 수업을 끝까지 완주하지 못한다(K-MOOC 이수율 13.1%). 교사는 수업을 하지만 그것이 닿았는지 모르고, 학생은 막히지만 말할 채널이 없으며, 운영자는 숫자만 보고 맥락을 모른다.

## 솔루션

**[프로젝트명]**([태그라인 TBD])는 "학생 피드백 수집 → AI 이해도 분석 → 교사 대시보드 → 다음 수업 초안 자동 생성"이라는 핵심 루프로 교사-학생 사이의 끊어진 피드백 루프를 AI로 복원하는 교육 플랫폼이다. 피드백 수집 단계에서는 게이미피케이션(Kahoot 스타일 퀴즈·게임) UX를 활용하여 학생의 자발적 참여를 유도한다.

| MVP 핵심 기능 | 설명 |
|----------|------|
| **학생 피드백 수집** | 수업 후 학생이 피드백을 제출. UX 레이어로 게이미피케이션(퀴즈 게임·리더보드·포인트)을 적용하여 자발적 참여 유도 — 핵심은 피드백 데이터 수집이지 게임 자체가 아님 |
| **AI 이해도 분석 + 교사 대시보드** | 수집된 피드백 데이터를 AI가 분석하여 이해도 %를 산출. "어떤 개념이 몇%나 이해됐는지" 교사에게 실시간으로 시각화 제공 |
| **다음 수업 초안 자동 생성** | 이해도 분석 결과를 기반으로 AI가 다음 수업의 보강 포인트·자료 초안을 자동 생성 (마크다운 1클릭 복사) |
| **(부가) 따봉 업/다운 피드백** | 교사·학생이 수업/콘텐츠에 대해 간단히 평가 → 프로젝트 지속 개선 루프 |

## 핵심 차별점

- **교사가 먼저 가치를 느끼는 설계**: ClassDojo 패턴 — 교사가 수업 중 피드백 수집(게임 형태)을 실행하고, 수업 후 AI 대시보드에서 학생 이해도를 즉시 확인
- **게이미피케이션은 피드백 수집의 UX 전략**: 강제 설문 대신 Kahoot 스타일 퀴즈 게임으로 학생이 "재미있어서" 자발적으로 참여 — 피드백 데이터를 자연스럽게 수집하는 방법
- **"피드백 → AI 분석 → 수업 개선" end-to-end**: Kahoot은 퀴즈 게임 후 AI 분석이 없고, Magic School AI는 학생 피드백 데이터가 없다. [프로젝트명]는 피드백 수집부터 다음 수업 초안까지 전 과정을 AI가 자동화

# ch1. 프로젝트 개요

## 1.1 프로젝트 배경

한국 교육 현장은 구조적 위기에 놓여 있다. 교사의 행정업무 시간은 **주당 6시간**으로 OECD 평균(3시간)의 **2배**이며, 교직 만족도는 **22.7%**에 불과하다. K-MOOC 평균 이수율은 **13.1%**로, 10명 중 9명이 온라인 수업을 끝까지 완주하지 못한다.

이 세 수치는 독립된 문제가 아니다. 하나의 악순환이다.

```
교사가 행정업무에 치여 → 학생에게 피드백할 시간이 없고
→ 피드백이 없으니 학생은 "나를 봐주는 사람이 없다"고 느끼며 이탈하고
→ 이탈은 사후에야 통계로 잡히니 교사도 운영자도 선제 개입이 불가능하고
→ 교사는 "내 가르침이 통했는지 알 수 없어" 보람을 잃고 번아웃된다
```

이 악순환의 핵심 고리는 **"수업 후 피드백 루프의 부재"**다. 교사는 수업을 하지만 그것이 닿았는지 모르고, 학생은 막히지만 말할 채널이 없다.

## 1.2 프로젝트 목표

**[프로젝트명]**([태그라인 TBD])는 "학생 피드백 수집 → AI 이해도 분석 → 교사 대시보드 → 다음 수업 초안 자동 생성"의 핵심 루프로 **교사-학생 사이에 끊어진 피드백 루프를 복원**하는 교육 플랫폼이다. 피드백 수집 단계에는 게이미피케이션 UX를 적용하여 학생의 자발적 참여를 유도한다.

| 목표 | 구체적 지표 |
|------|-----------|
| 교사의 수업 임팩트 가시화 | 학생 피드백 데이터 기반 AI 이해도 분석 결과를 대시보드로 실시간 제공 |
| 교사의 수업 준비 시간 절약 | AI가 이해도 분석 후 다음 수업 보강 포인트·초안 자동 생성 (AI 기반 31% 시간 단축 효과 기대) |
| 학생의 자발적 참여 유도 | 게이미피케이션 UX(퀴즈 게임·리더보드) → 학생이 "하고 싶어서" 피드백에 참여하는 구조 |

## 1.3 핵심 가치 제안 (Value Proposition)

> **"학생 피드백을 게이미피케이션 UX로 자연스럽게 수집하고, AI가 이해도를 분석하여 교사의 다음 수업을 자동으로 개선한다."**

기존 에듀테크가 **학습 효율**에 집중하는 동안, [프로젝트명]는 **"피드백 수집 → AI 이해도 분석 → 수업 개선"이라는 end-to-end 루프**에 집중한다. 게이미피케이션은 이 루프의 첫 단계(피드백 수집)를 재미있게 만드는 UX 전략이다. 학생은 즐기며 참여하고, 교사는 AI 인사이트로 수업을 개선하는 선순환 구조를 만든다.

# ch2. 페인포인트 분석

## 2.1 본질적 문제 정의: 피드백 루프의 단절

교육의 본질은 정보 전달이 아니라 **"한 사람이 다른 사람의 성장을 지켜보고, 그 사람이 지켜봄을 느끼는 관계"**다. 좋은 교사를 떠올릴 때 우리가 기억하는 것은 교과 내용이 아니라, "그 사람이 나를 봐줬다"는 경험이다.

그러나 지금, 이 관계가 구조적으로 무너지고 있다.

```
교사의 가르침 ─── ✕ ───▶ 학생에게 닿았는지 알 수 없다
     "이해했니?"              "모르겠는데 말 못 해"

학생의 막힘 ─── ✕ ───▶ 교사에게 보이지 않는다
     "여기서 막혔어요"         "30명 중 누가 막혔는지 몰라"

현장의 진실 ─── ✕ ───▶ 운영자에게 들리지 않는다
     "이 수업이 좋았다"        "수료율 숫자만 본다"
```

### 2.2.2 교사 번아웃 및 교직 회의

2023년 기준 교사의 **24.9%가 가벼운 우울**, **38.3%가 심한 우울** 증상을 겪고 있다 (합산 63.2%). 교직 만족도는 **22.7%**에 불과하며, 교사 78%는 교권회복 4법 시행 이후에도 근무 여건이 개선되지 않았다고 응답했다.

교사가 진짜 원하는 것은 업무 자동화가 아니다. **"내 가르침이 누군가에게 닿고 있다"는 증거**다.

> 출처: 교사노동조합연맹 2024 스승의 날 설문 ([교육플러스](https://www.edpl.co.kr/news/articleView.html?idxno=12696))

### 2.2.3 학습자 수준 파악 및 개인 피드백의 한계

30명 이상의 학급에서 교수자가 각 학습자의 이해도와 진도를 실시간으로 파악하고 개별 피드백을 제공하기는 사실상 불가능하다. 피드백은 평균 **2주** 지연되며, 그 시점이면 이미 다음 단원으로 넘어간 상태다.

> 출처: doc 01 §1-2, [에듀인뉴스](https://www.eduinnews.co.kr/news/articleView.html?idxno=38442)

## 2.3 학생 관점 페인포인트

### 2.3.1 학습 동기 저하 및 낮은 완강률

K-MOOC 서비스 시작(2015년) 이후 평균 이수율은 **13.1%**에 불과하다. 온라인 수업이 면대면 수업에 비해 집중도가 낮다고 응답한 비율이 **62%**에 달한다. 학생이 진짜 원하는 것은 AI 챗봇이 아니다. **"내가 어디서 막혀있는지 누군가 알아채주는 것"**이다.

> 출처: KCI ([학습동기와 이수율 상관관계](https://www.kci.go.kr/kciportal/ci/sereArticleSearch/ciSereArtiView.kci?sereArticleSearchBean.artiId=ART002556079)), [인키움 2024](https://www.inkium.com/column/?bmode=view&idx=17951014)

### 2.3.2 즉각적 질문·답변 채널 부재

온라인 수업 환경에서 학습자가 즉시 질문하고 답변을 받을 수 있는 채널이 부족하다. 교수자와의 비동기 소통으로 인해 이해 공백이 누적된다. 30명 중 한 명인 학생은 손 들어 질문하면 수업 흐름을 끊고, 모른다고 말하면 뒤처진 사람이 된다.

> 출처: doc 01 §2-3, [한국경제](https://www.hankyung.com/article/202206165852i)

### 2.3.3 학습 성과 피드백의 부재

과제 피드백이 지연되거나 형식적으로 처리되는 경우가 많아, 학습자는 자신의 실제 역량 수준을 파악하기 어렵다. AI 자동 채점 시 마킹 속도 80% 향상이 가능하다는 연구 결과가 있다.

> 출처: [ResearchGate — Automated Grading and Feedback Systems](https://www.researchgate.net/publication/387254488_Automated_Grading_and_Feedback_Systems_Reducing_Teacher_Workload_and_Improving_Student_Performance)

# ch4. 경쟁 환경 분석

## 4.1 경쟁 구도 개요

현재 AI 교육 시장의 경쟁자들은 크게 두 방향으로 나뉜다:

1. **학습 최적화 방향** (대다수): AI를 학습 효율·점수 향상에 활용. 교사-학생 관계에는 무관심.
2. **교사 보조 도구 방향** (소수): AI로 교사 업무를 경감. 관계 "데이터"는 보지만 관계 "회복"까지는 가지 않음.

**[프로젝트명]의 포지셔닝**: "AI 활용 깊이 + 관계 중심 생태계"의 교차점 — 현재 **명확한 공백(White Space)**이다.

## 4.3 주요 경쟁사 심층 분석

### 4.3.1 클래스팅 AI — 국내 공교육 1위, 그러나 "관계"는 미흡

- **규모**: 누적 890만 명, 전국 4,600개+ 학교, 47개국 서비스
- **2025 업데이트**: AI 국어 쓰기 평가, 실시간 모니터링, AI 수능 서비스(B2C) 진출
- **강점**: 국내 공교육 점유율 1위, 프라이빗 LLM 보유, 교사-학생 소통 기능 내장
- **약점**: 학습 관리 + AI 평가 중심 → 수업 피드백 루프, 교사 임팩트 가시화 기능 부재
- **[프로젝트명]와의 차이**: 클래스팅은 "소통 + 학습 관리" 도구. [프로젝트명]는 수업 후 피드백이 다음 수업을 실제로 바꾸는 루프를 AI로 자동화.

> 출처: 클래스팅 공식 블로그 ([2025 업데이트](https://blog.classting.com/classting-ai-2025-updates/)), doc 07 §A-1, research-supplement §5

### 4.3.2 Magic School AI — 글로벌 교사 도구 1위, 그러나 "관계"는 없다

- **규모**: 600만 명+ 교육자 (미국 K-12 교사 전체 수 초과), 80개+ 교사 전용 도구
- **성과**: 교사 주당 **10시간** 절약, 일간 활성 사용자 비율 70%, 누적 퀴즈 1,000만 개 생성
- **강점**: teacher-first 설계의 교과서, PLG(Product-Led Growth) 성공
- **약점**: 교사 생산성 도구에 한정. 학생 피드백, 교사-학생 관계 연결 기능 없음.
- **[프로젝트명]와의 차이**: Magic School은 교사가 "혼자 쓰는" 도구. [프로젝트명]는 교사-학생 사이의 피드백 루프를 AI가 연결.

> 출처: research-supplement §2-1, [MagicSchool.ai](https://www.magicschool.ai), [Wifitalents 2026](https://wifitalents.com/magicschool-statistics/)

### 4.3.3 Gradescope — 채점 자동화의 정석

- **규모**: 2,600개 대학, 14만 명 강사, 320만 명 학생
- **핵심 설계**: AI가 유사 답안을 자동 그룹화 → 교사가 그룹 단위로 한 번만 채점 (30~50% 시간 절감)
- **교훈**: 교사의 판단을 살리되, 반복 노동을 제거하는 구조 → [프로젝트명]가 참고할 설계 원칙
- **[프로젝트명]와의 차이**: Gradescope는 채점 효율화 도구. [프로젝트명]는 게이미피케이션 + AI 인사이트 플랫폼.

> 출처: doc 09 §2.2, [Gradescope](https://www.gradescope.com/)

### 4.3.4 Khanmigo — AI 튜터의 표준, 그러나 한국 부재

- **규모**: Khan Academy 전체 이용자 수억 명 기반, GPT-4 기반 소크라테스식 튜터
- **가격**: 교사 무료(44개국), 학부모 월 $4
- **강점**: 비영리 신뢰도, 교사 도구 + 학생 튜터 통합, 저렴
- **약점**: 한국어 미지원, 한국 교육과정 미반영, 교사-학생 관계 데이터화 기능 없음

> 출처: doc 02 §2-1, doc 07 §A-2

### 4.3.5 한국 AI 디지털교과서 — "의무화 실패"의 교훈

2025년 3월 도입된 76종 AI 디지털교과서(AIDT)는 **4개월 만에 사실상 폐기**되었다.
- 초등학교 채택률 30% 미만, 참여 학교 절반 이상 10월까지 이탈
- 기술 오류 다수, 교사·학생 시간 부담 증가
- 2025년 10월 "보충 자료"로 재분류 — 교사 선택 사항으로 강등

**핵심 교훈**:
1. "의무화"가 아니라 "자발적 채택 → 동료 추천 → 바이럴"이 정답
2. 교사가 별도 학습 없이 즉시 가치를 느낄 수 있는 설계가 관건
3. B2G 의존 리스크 → B2B + B2C 병행 전략 필요

> 출처: [Futurism](https://futurism.com/future-society/south-korea-ai-textbook), [Rest of World](https://restofworld.org/2025/south-korea-ai-textbooks), [Korea Herald](https://www.koreaherald.com/article/10546695)

## 6.0 학생 참여 UX 전략 — 게이미피케이션이 피드백 수집을 재미있게 만드는 이유

> **배경:** [프로젝트명]의 핵심 루프에서 첫 단계는 "학생 피드백 수집"이다. 강제 설문으로는 자발적 참여를 기대할 수 없다. 게이미피케이션(Kahoot 스타일 퀴즈 게임)은 이 피드백 수집 단계를 재미있게 만드는 UX 레이어다. [프로젝트명]의 학생 타겟(Secondary Persona)은 10대이므로, 10대가 "재미있다"고 느끼는 UX 패턴을 분석하고 게이미피케이션 설계 근거를 제시한다.

### 게이미피케이션의 심리적 메커니즘 (SDT 기반)

자기결정이론(Self-Determination Theory)에 따르면 인간의 내재적 동기는 3가지 기본 심리 욕구에서 나온다:

| 심리 욕구 | 정의 | [프로젝트명] 충족 방식 |
|---------|------|-------------------|
| **유능감(Competence)** | "내가 잘하고 있다"는 느낌 | 즉각 정답 피드백 + 점수 시각화 |
| **자율성(Autonomy)** | 자발적 선택, 강제 아님 | 게임이니까 "하고 싶어서" 참여 |
| **관계성(Relatedness)** | 또래와 연결되는 느낌 | 교실 전체 동시 참여 + 리더보드 경쟁 |

> **연구 근거:** 게이미피케이션은 학생의 내재적 동기, 자율성, 관계성에 긍정적 효과를 보인다 (Springer 2024 메타분석). 특히 청소년 대상 연구에서 거의 모든 연구에서 동기 증가가 확인되었다.
>
> 출처: [Gamification enhances intrinsic motivation — Springer 2024](https://link.springer.com/article/10.1007/s11423-023-10337-7), [Gamification and motivation in adolescents — Frontiers 2025](https://www.frontiersin.org/journals/psychology/articles/10.3389/fpsyg.2025.1575104/full)

### 왜 [프로젝트명]의 게이미피케이션이 10대에게 먹히는가

10대 사용자는 다음 조건이 갖춰질 때 자발적으로 참여한다:

1. **즉각성**: 로딩·설치·가입 없이 바로 시작 → QR 스캔만으로 즉시 접속
2. **경쟁**: 또래와 실시간 순위 비교 → 교실 리더보드
3. **부담 없음**: 틀려도 괜찮은 분위기 → 게임이니까 실패가 재미의 일부
4. **가시성**: 내 성과가 즉시 화면에 보임 → 실시간 점수·순위 업데이트
5. **사회적 맥락**: 친구들과 함께하는 경험 → 교실 전체가 동시에 참여

K-MOOC 이수율 13.1%의 핵심 원인은 혼자, 강제로, 지루하게 학습하는 구조다. [프로젝트명]는 이를 함께, 자발적으로, 게임처럼으로 뒤집는다.

## 6.1 핵심 가치 제안

> **"피드백 수집 → AI 이해도 분석 → 교사 대시보드 → 다음 수업 초안 자동 생성" — 게이미피케이션은 이 루프의 첫 단계를 재미있게 만드는 UX 전략이다.**

[프로젝트명]의 핵심 가치는 피드백 루프 전체를 자동화하는 것이다. 기존 에듀테크가 "학습 효율"에 집중하는 동안, [프로젝트명]는 **"교사의 가르침이 학생에게 닿았는지 알 수 없고, 학생의 막힘이 교사에게 보이지 않는"** 피드백 단절이라는 교육의 본질적 문제를 해결한다 (narrative-spine.md §1). 게이미피케이션(Kahoot 스타일 퀴즈 게임)은 피드백 수집 단계에서 학생의 자발적 참여를 이끄는 UX 접근이다 — 강제 설문이 아닌 게임이기에 학생이 "하고 싶어서" 피드백을 제출한다.

Kahoot은 게이미피케이션으로 학생 참여 문제를 해결했지만 AI 분석이 없다. Magic School AI는 교사 생산성을 높이지만 학생 참여 데이터가 없다. [프로젝트명]는 이 둘을 결합한다 — **"재미있는 참여 + AI 인사이트"**라는, 현재 시장에 없는 조합.

> **서사 원칙 (narrative-spine.md §6):** "관계 회복"은 프레이밍이고, "재미있는 참여"는 훅(hook)이다. 학생을 움직이는 즉각적 동기는 게임의 재미이지만, [프로젝트명]가 다른 도구와 차별화되는 포지셔닝은 참여 데이터 기반 관계 회복이다. "재미로 들어와서 관계 회복으로 남는다."

## 6.3 Aha Moment — "교사가 [프로젝트명]를 쓰겠다고 결심하는 순간"

> **"학생들이 퀴즈 게임에 열중하며 참여하는 모습을 보고, 수업 후 AI 대시보드에서 '이 개념 정답률 33% — 보강 필요'라는 데이터를 확인하는 순간."**

**왜 이것이 Aha Moment인가:**

1. **즉각적 참여 증거**: 교사가 퀴즈 게임을 실행하는 순간, 학생들이 스마트폰을 들고 열중하는 모습이 눈앞에 보인다. "학생들이 진짜 참여하고 있다!"는 시각적 증거.
2. **AI 인사이트의 구체성**: 게임이 끝나고 대시보드를 열면 "도함수 그래프 해석 정답률 33% — 학생 67%가 이 개념에서 어려움"이라는 구체적 데이터. 막연한 직감이 데이터로 전환.
3. **행동 유발**: 인사이트를 확인한 교사는 즉시 "수업 개선 제안" 버튼을 누른다. AI가 다음 수업 보강 포인트를 자동 생성 → 교사는 내일 수업에 바로 적용.

**Aha Moment의 시간 설계:**

```
수업 마지막 10분: 퀴즈 게임 실행
  → 학생들 QR 접속 + 게임 참여 (10분)
  → 게임 중 실시간 리더보드로 학생 열중 (← 여기서 이미 교사는 "이거 좋다!" 체감)
수업 종료 후:
  → AI 대시보드에서 참여 데이터 기반 인사이트 확인 (30초)
  → "수업 개선 제안" 클릭 → AI가 다음 수업 보강 포인트 생성 (10초)
  → 완료. 교사는 수업 중 학생 참여를 직접 보고, 수업 후 AI 인사이트까지 얻는다.
```

**Kahoot과의 결정적 차이:**
Kahoot은 게임이 끝나면 끝이다. 교사가 결과를 직접 분석하고 다음 수업에 어떻게 반영할지 스스로 판단해야 한다. [프로젝트명]는 게임 종료 후 AI가 자동으로 "어떤 개념이 어려웠는지, 다음 수업에서 무엇을 보강해야 하는지"까지 제안한다. **Kahoot에서 멈추는 곳에서 [프로젝트명]가 시작한다.**

**에듀테크 온보딩 창(window) 내 달성:**
esparklearning(2024) 설문에 따르면 **교사의 66%가 30분 이내에 제품 채택 여부를 결정**한다. [프로젝트명]의 Aha Moment는 첫 퀴즈 게임 실행 중에 즉시 발생하므로(학생들이 열중하는 모습), 이 임계 시간 안에 충분히 가치를 전달한다 (출처: docs/background/11).

## 6.4 기존 솔루션과의 차별점

| 차별화 축 | Magic School AI | Kahoot | 클래스팅 AI | [프로젝트명] |
|-----------|----------------|--------|------------|-------|
| **핵심 기능** | 수업 준비·워크시트 생성 | 실시간 퀴즈 게임 | AI 학습관리(LMS) + 평가 | **게이미피케이션 + AI 인사이트 + 수업 개선** |
| **AI의 역할** | 콘텐츠 생성 도구 | **없음** (집계·시각화만) | 적응형 학습 + 평가 분석 | 참여 데이터 AI 분석 + 인사이트 + 수업 개선 제안 |
| **학생 참여 동기** | 없음 (교사 전용) | **게임의 재미** (리더보드, 경쟁) | 의무적 (앱 설치 + 학습) | **게임의 재미** + AI가 데이터를 자연스럽게 수집 |
| **교사 가치 발생 시점** | 수업 준비 시 | 수업 중 실시간 | 데이터 축적 후 | 수업 중 (학생 참여 목격) + 수업 후 (AI 인사이트) |
| **데이터 → 수업 개선** | 연결 없음 | **교사가 직접 해석** | 학습 데이터 제공, 수업 개선은 교사 몫 | **AI가 분석 + 개선 제안까지 end-to-end** |
| **규모** | 600만+ 교육자 (2025) | 누적 90억 명 참가, 200개국 | 890만 가입, 4,600개 학교 | MVP (신규) |

**핵심 공백:** Kahoot은 **게이미피케이션으로 참여를 높이지만 AI 분석이 없다**. Magic School AI는 **AI로 교사 생산성을 높이지만 학생 참여 데이터가 없다**. 클래스팅은 학습 관리 도구이지만 수업 피드백 루프가 없다. **"Kahoot의 재미있는 참여 + AI 인사이트 분석 + 자동 수업 개선 제안"을 결합한 도구는 현재 시장에 없다.** [프로젝트명]는 이 공백을 점유한다 (출처: docs/background/07 포지셔닝 공백 분석, research-supplement.md §5).

> **Kahoot과의 비교:**
> Kahoot은 게이미피케이션 교육의 표준이다 (누적 참가자 90억 명, 500M+ 게임 실행, 200개국, Fortune 500의 70% 사용). 그러나 Kahoot의 한계는 명확하다: (1) AI 분석 없음 — 교사가 결과를 직접 해석해야 함, (2) 개인화 인사이트 없음 — 어떤 학생이 어떤 개념에서 뒤처지는지 알 수 없음, (3) 수업 개선 연결 없음 — 게임이 끝나면 끝. [프로젝트명]는 Kahoot이 멈추는 곳에서 시작한다.
> (출처: Kahoot 공식 — https://kahoot.com/company/, Kahoot Annual Report 2024)

> **참고 — AI 디지털교과서 실패의 교훈 (research-supplement.md §3):**
> 2025년 한국 정부의 AI 디지털교과서(AIDT) 76종이 4개월 만에 사실상 폐기되었다. 핵심 실패 원인: (1) 급진적 전면 의무화 → 교사 저항, (2) 기술 오류 + AI 결과물 검수 부담 → 업무 오히려 증가, (3) 교사가 별도 학습 없이 즉시 가치를 느낄 수 없는 설계.
> [프로젝트명]는 이 교훈을 정면으로 반영한다: **자발적 채택 모델** (의무화 아님), **게임이라서 즉시 가치 체감** (학생 참여가 눈에 보임), **검수 부담 제로** (수치 중심 출력, 낮은 위험도 자동화).

| 항목 | 내용 |
|------|------|
| **개요** | 기능 1에서 수집된 피드백 데이터를 AI가 분석하여 학생 이해도 %를 산출하고, 교사에게 "어떤 개념이 몇%나 이해됐는지" 실시간으로 시각화하여 제공 |
| **실시간 집계** | 게임 진행 중 실시간으로 정답률, 응답 시간, 참여율이 대시보드에 반영 (Supabase Realtime 활용) |
| **AI 핵심 발견 자동 생성** | 게임 종료 후 "분석하기" 버튼 클릭 → 10~15초 내 AI가 분석 결과 출력: (1) 이번 수업 핵심 취약 개념 Top 3 (정답률 기반), (2) 학생들이 강한 개념 vs 약한 개념, (3) 다음 수업 권장 보강 포인트 |
| **시각화** | 문항별 정답률 차트, 개념별 이해도 히트맵, 취약 개념 하이라이트 |
| **교사 행동 유도** | 인사이트 확인 후 "수업 개선 제안" 버튼으로 자연스럽게 기능 3으로 연결 |
| **예상 구현 공수** | 1.5일 (Next.js 대시보드 + Supabase Realtime + Claude API 프롬프트 엔지니어링) |

**AI 프롬프트 설계 원칙:**
1. **수치 중심**: "도함수 그래프 해석 정답률 33% — 학생 67%가 이 개념에서 어려움"처럼 교사가 즉각 판단 가능한 형태
2. **교육 맥락 결합**: 과목·학년·단원 정보를 프롬프트에 포함하여 도메인 특화 인사이트 생성
3. **행동 가능한 제안**: "다음 수업에서 도함수의 시각적 해석을 먼저 다루세요" 같은 구체적 다음 행동 제시
4. **검수 부담 최소화**: RSC(2024) 연구 경고 — AI 결과물 검수 부담이 채택을 방해. 낮은 위험도의 출력물(수치 요약, 정답률)만 자동화

**Kahoot과의 결정적 차이:**
Kahoot은 게임 결과를 보여주지만 **AI 분석이 없다**. 교사가 직접 결과를 해석하고 다음 수업에 어떻게 반영할지 판단해야 한다. [프로젝트명]는 AI가 자동으로 "어떤 개념이 어려웠는지, 누가 뒤처지는지, 다음 수업에서 무엇을 보강해야 하는지"를 분석하여 제공한다.

**기술 구현:**
```
학생 게임 참여 데이터
  → Supabase PostgreSQL에 저장 (문항별 정답/오답, 응답 시간, 점수)
  → Supabase Realtime으로 교사 대시보드에 실시간 반영
  → 게임 종료 후 교사가 "분석하기" 클릭
  → Next.js API Route에서 Claude API 호출
    - 시스템 프롬프트: 과목, 학년, 단원 맥락 + 출력 형식 지정
    - 사용자 프롬프트: 문항별 정답률 + 응답 시간 + 오답 패턴
  → 구조화된 JSON 응답 → 대시보드에 렌더링
```

(출처: docs/background/04 — Next.js API Routes + Vercel 배포 + 환경변수 기반 API Key 보안)

### 기능 3: 다음 수업 초안 자동 생성 (AI Class Improvement Suggestions)

| 항목 | 내용 |
|------|------|
| **개요** | 기능 2의 이해도 분석 결과를 바탕으로 AI가 다음 수업의 보강 포인트, 추천 활동, 슬라이드 초안을 자동 생성 |
| **제안 구성** | (1) "지난 수업 퀴즈에서 약했던 개념" — 취약 개념 재설명 포인트 (2) "추천 보강 활동" — 해당 개념 강화를 위한 퀴즈/활동 제안 (3) "다음 수업 오프닝 슬라이드" — AI 제안 기반 마크다운 |
| **출력 형식** | 마크다운 텍스트 → 교사가 1클릭으로 복사하여 PPT/구글 슬라이드에 붙여넣기 |
| **교사 수정 권한** | 교사가 원하면 직접 수정 가능하나, 수정 없이 바로 사용해도 무방한 수준의 품질 목표 |
| **예상 구현 공수** | 0.5일 (Claude API 생성 + 복사 버튼 UI) |

**핵심 가치:**
이 기능이 [프로젝트명]의 결정적 차별점이다. Kahoot은 게임 결과를 보여주지만, **"피드백 데이터를 다음 수업에 어떻게 반영할지"는 전적으로 교사의 몫**이다. [프로젝트명]는 이 격차를 AI로 자동화한다 — "피드백 수집 → AI 이해도 분석 → 다음 수업 초안 자동 생성까지 end-to-end 파이프라인" (출처: docs/background/11).

**doc 09 교훈 반영:**
- Gradescope: AI가 유사 답안을 자동 그룹화 → 교사는 "그룹 단위로 한 번만 채점". **교사의 판단은 살리되, 반복 노동을 제거.**
- 동일 원칙 적용: AI가 수업 개선안을 생성 → 교사는 "검토 후 적용". 교사의 교육적 판단은 살리되, 수업 준비의 반복 작업을 제거.

> **I WANT TO** 이차함수 수업 마지막 10분에 퀴즈 게임을 실행하여 학생들의 이해도를 파악하고 싶다,
> **SO THAT** 학생들이 재미있게 참여하면서 동시에 이해도 데이터를 얻을 수 있다.
>
> **수용 기준:**
> - 수업 중 QR 코드를 화면에 띄우면 학생들이 스마트폰으로 게임에 접속
> - 실시간 리더보드에 학생 순위가 표시되며 학생들이 열중
> - 게임 종료 후 "분석하기" 클릭 시 "도함수 그래프 해석 정답률 33% — 보강 필요" 같은 AI 인사이트 표시

### 스토리 2: AI 수업 개선 제안 받기

> **AS** 영어 교사 박 선생님,
> **I WANT TO** 지난 수업 퀴즈 결과를 반영한 다음 수업 개선 제안을 자동으로 받고 싶다,
> **SO THAT** 수업 준비 시간을 줄이면서도 학생 약점을 보강하는 수업을 할 수 있다.
>
> **수용 기준:**
> - 대시보드에서 "수업 개선 제안" 클릭
> - 10초 내 보강 포인트 + 슬라이드 초안 생성
> - "지난 시간 퀴즈에서 관계대명사 정답률이 낮았습니다. 오늘 다시 다룹니다" 같은 맥락 반영
> - 1클릭 복사 → PPT/구글 슬라이드에 붙여넣기 가능

### 스토리 3: 수업 개선 추적

> **AS** 학원 수학 강사 이 선생님,
> **I WANT TO** 같은 과목의 이전 수업과 이번 수업 퀴즈 결과를 비교하고 싶다,
> **SO THAT** 내 수업이 실제로 개선되고 있는지 확인할 수 있다.
>
> **수용 기준:**
> - 대시보드에서 과목별 수업 이력 조회 가능
> - "지난주 대비 이차함수 정답률 33% → 65%로 향상" 표시
> - 시간에 따른 개념별 정답률 변화 추이 시각화

### 스토리 4: 운영자 강좌 모니터링

> **AS** 학원장 박 원장,
> **I WANT TO** 5명 강사의 강좌별 피드백 트렌드를 한눈에 보고 싶다,
> **SO THAT** 문제가 있는 강좌를 조기에 발견하고 강사를 지원할 수 있다.
>
> **수용 기준:**
> - 운영자 대시보드에서 강좌별 피드백 요약 1페이지 확인
> - "A 강좌: 3주차부터 '진도가 빠르다' 피드백 급증" 알림
> - 강사별 피드백 응답률 비교

### 스토리 5: 학생의 게임 참여

> **AS** 수업 중 질문하기 부끄러운 고등학생 수진이,
> **I WANT TO** 재미있는 퀴즈 게임에 참여하면서 자연스럽게 내 이해도를 선생님에게 전달하고 싶다,
> **SO THAT** 다음 수업에서 내가 어려워한 부분을 다시 설명받을 수 있다.
>
> **수용 기준:**
> - QR 스캔 후 로그인 없이 바로 게임 참여
> - 리더보드에서 실시간 순위 경쟁 — 재미있어서 적극 참여
> - 다음 수업 시작 시 "지난 시간 퀴즈에서 많이 틀린 부분을 오늘 다시 다룹니다" 확인
> - 따봉 피드백으로 "이 수업 좋았다/아쉬웠다" 간단히 표현 가능

## 8.4 데모 시나리오 (3분 시연 흐름)

> **목표:** 심사위원 앞에서 라이브로 게이미피케이션 → AI 인사이트 → 수업 개선의 전체 루프를 시연하여 "재미있는 참여가 수업을 바꾼다"를 체험하게 한다.

**스크립트:**

### Scene 2: 퀴즈 게임 라이브 시연 (0:30 ~ 1:40)

**화면:** 교사 대시보드 → 퀴즈 게임 시작 → QR 코드 표시

**스크립트:**
"지금 심사위원 여러분이 학생이라고 가정하겠습니다. 이 QR을 스캔해서 게임에 참여해주세요."

3. 심사위원들이 경쟁하며 참여 — "지금 1등은 OO님입니다!"
4. 게임 종료 → 최종 리더보드 표시

**스크립트:** "학생들은 이걸 '설문'이 아니라 '게임'으로 인식합니다. Kahoot이 90억 명을 참여시킨 이유입니다."

**백업:** 네트워크 불안정 시 사전 녹화 영상 전환 (3초 내)

### Scene 3: AI 인사이트 생성 (1:40 ~ 2:20)

**화면:** 대시보드 → "분석하기" 버튼

**스크립트:**
"게임이 끝났습니다. 여기서 Kahoot은 멈춥니다. [프로젝트명]는 여기서 시작합니다. AI가 분석합니다."

**액션:**
1. "분석하기" 클릭
2. 10~15초 로딩 (스피너 표시)
3. AI 인사이트 출력:
   - "핵심 발견 1: 도함수 그래프 해석 문제 정답률 33% — 학생 67%가 이 개념에서 어려움"
   - "핵심 발견 2: 시각적 예시 관련 문제 정답률 83%로 높음"
   - "권장: 다음 수업에서 도함수의 시각적 해석을 먼저 다루세요"

**스크립트:** "교사는 30초 만에 '학생들이 뭘 어려워했고, 뭐가 효과적이었는지'를 데이터로 파악했습니다."

### Scene 4: AI 수업 개선 제안 (2:20 ~ 3:00)

**화면:** "수업 개선 제안" 버튼

**액션:**
1. 버튼 클릭 → AI가 다음 수업 보강 포인트 + 슬라이드 초안 즉시 생성
2. 화면에 표시: "지난 수업 퀴즈에서 도함수 그래프 해석 정답률이 낮았습니다. 시각적 예시를 먼저 다루는 것을 추천합니다. 추천 활동: 그래프 매칭 퀴즈."
3. "복사" 버튼 → 교사가 내일 수업 PPT에 바로 붙여넣기

**마무리 스크립트:**
"학생은 재미있는 게임을 했고, 교사는 AI 인사이트로 수업을 개선합니다. 학생은 다음 수업에서 '선생님이 우리가 틀린 부분을 보강해줬구나'를 경험합니다. **Kahoot이 멈추는 곳에서 [프로젝트명]가 시작합니다.**"

## 9.3 Strengths (강점)

### S1. 11건 리서치 기반의 증거 중심 기획

[프로젝트명]의 모든 설계 결정은 docs/background/ 01~11의 체계적 리서치에 근거한다. 특히 doc 09(스타트업 성공/실패 사례)와 doc 11(솔루션 비판적 재검토)를 통해 **"교사가 먼저 가치를 느끼는 설계"**라는 핵심 원칙을 도출했다. 공모전 심사 기준 중 "기획력 및 실무 접합성"에서 높은 점수를 기대할 수 있는 구조다.

**근거:** doc 03 수상작 분석 — 수상 프로젝트 6/6이 "명확한 페인포인트 타겟"을 갖추고 있었으며, "실증 데이터 또는 실제 사용자"를 제시한 프로젝트가 4/6로 높은 수상률을 보임.

### S2. 교사가 먼저 가치를 느끼는 설계 (ClassDojo + Kahoot 패턴) + AI 분석 루프

성공한 에듀테크의 불변 법칙: **"교사가 직접 통제하고, 학생은 재미있어서 자발적으로 참여"** (doc 09 §5.2). [프로젝트명]는 이 패턴을 정확히 따른다:
- 교사는 피드백 수집(게이미피케이션 UX)을 실행 → 학생들이 열중하는 모습이 즉각적 가치
- 학생은 게임 형태이므로 자발적으로 피드백 참여 → 강제 설문이 아니라 게임이므로 지속성 확보
- 피드백 수집 후 AI 이해도 분석 + 다음 수업 초안이 교사에게 즉각적 시간 절약 제공

**근거:** Kahoot은 게이미피케이션으로 누적 90억 명 참가, 200개국 사용 — 재미있으면 참여한다는 것을 증명. ClassDojo는 교사가 직접 통제하는 도구로, 론칭 12주 만에 35,000교실 확보. 미국 K-8 학교의 95%+ 점유 (doc 09 §2.1).

### S3. 단순 구조의 높은 완성도 가능성

MVP 기능이 3개뿐이며 기술 스택이 단순(Next.js + Supabase + Claude API + Vercel)하여, 1주 내 완성도 높은 라이브 데모 구현이 가능하다.

**근거:** doc 03 실패 프로젝트 공통 패턴 — "기능 과다, 완성도 낮음". 수상작 공통 패턴 — "1~3개 기능 집중, 완성도 높음". DEV Community(2024) 동일 스택 해커톤 주말 완성 사례 (doc 11).

### S4. "피드백 수집 → AI 이해도 분석 → 수업 개선" end-to-end 루프 포지셔닝 공백 점유

Kahoot은 게이미피케이션으로 참여를 높이지만 피드백 데이터의 AI 분석이 없다. Magic School AI는 AI로 교사 생산성을 높이지만 학생 피드백 데이터가 없다. **"게이미피케이션 UX로 피드백을 수집하고 AI가 이해도를 분석하여 다음 수업까지 자동 개선하는" 도구는 현재 시장에 없다.** Kahoot 누적 90억 명 참가로 게이미피케이션의 참여 효과는 증명되었고, [프로젝트명]는 이를 피드백 수집의 UX 레이어로 활용하며 AI 분석·수업 개선까지 더한다. 500팀 중 유사 접근 예상 비율: 3~5% (doc 08 §파트B).

> 출처: Kahoot 공식 (https://kahoot.com/company/), Kahoot Annual Report 2024

## 9.4 Weaknesses (약점)

### W1. 소규모 팀 — 개발·마케팅·영업 역량의 한계

1인 또는 소규모 팀으로 MVP를 개발하므로, 공모전 이후 서비스 운영·확장에 필요한 인력이 부족하다. 특히 B2B 영업, 교사 온보딩 지원, 지속적 기능 개발을 동시에 수행하기 어렵다.

**완화 방안:** MVP는 풀스택 프레임워크(Next.js)로 단일 코드베이스 유지. 공모전 이후 수상 시 투자·인력 확보 경로 마련. 초기에는 PLG(Product-Led Growth)로 교사 자발적 확산에 의존.

### W2. Kahoot 대비 기능 차별화 입증 필요

Kahoot은 이미 누적 90억 명 참가, 200개국 사용이라는 압도적 규모를 가지고 있다. [프로젝트명]의 핵심 전제는 "Kahoot에 없는 AI 인사이트가 교사에게 충분한 추가 가치를 제공한다"는 것이다. 그러나 **교사가 Kahoot의 기본 결과 화면 대신 AI 인사이트에 추가 비용/전환 비용을 지불할 의향이 있는지는 검증되지 않았다.**

**게이미피케이션의 참여 지속성 근거:**

Kahoot의 성공은 게이미피케이션이 학생 참여 지속성 문제를 해결할 수 있음을 증명한다. 강제 설문(60초 피드백)과 달리 게임은 "재미있어서" 참여하므로 지속성이 높다.

| 상황 | 참여율/지속성 |
|------|-------------|
| Kahoot 누적 참가자 | **90억 명** (2024) |
| Kahoot 실행된 게임 수 | **500M+** |
| Kahoot 사용 국가 | **200개국** |
| Fortune 500 사용 비율 | **70%** |
| 강제 설문 대비 게이미피케이션 참여율 | 유의미하게 높음 |

> 출처: Kahoot 공식 (https://kahoot.com/company/), Kahoot Annual Report 2024

**완화 방안:**
1. **게이미피케이션으로 참여 지속성 확보**: 강제 설문이 아닌 게임이므로 학생이 "하고 싶어서" 참여 — Kahoot이 증명한 패턴
2. **AI 인사이트의 명확한 추가 가치**: Kahoot에서는 교사가 직접 결과를 해석해야 하지만, [프로젝트명]는 AI가 자동으로 "어떤 개념이 약한지, 다음에 무엇을 보강할지"까지 제안
3. **미참여 시 fallback**: 과거 누적 게임 데이터 기반 인사이트 제공 — 당일 참여가 적어도 교사 가치 유지
4. **포인트 누적 + 리더보드**: 장기적으로 학생의 포인트가 누적되어 지속 참여 동기 부여

### W3. 수익 모델 미검증

SaaS 구독 모델을 계획하고 있으나, 교사/학교의 실제 지불 의향(WTP)은 검증되지 않았다. 한국 공교육 시장의 예산 구조상 개별 교사가 유료 도구를 결제하는 것이 일반적이지 않다.

**완화 방안:** 교사 개인 무료 → 학교/학원 유료(B2B) 전환 모델. 초기에는 무료 사용으로 PMF 검증에 집중.

### W4. 브랜드 인지도 제로

신규 서비스로서 교사 커뮤니티에서의 인지도가 전무하다. Magic School AI(600만+ 교육자), 클래스팅(890만 가입)과 비교하면 출발점의 격차가 크다.

**완화 방안:** PLG + 교사 커뮤니티(인디스쿨 등) 타겟 바이럴. ClassDojo 초기 전략과 동일 — 교사가 동료 교사에게 자발적으로 전파하는 구조 설계.

## 9.5 Opportunities (기회)

### O1. AI 교육 시장 CAGR 31.2% — 구조적 성장

글로벌 AI 교육 시장은 2025년 USD 7~8.3B에서 2030년 USD 32~41B으로 성장 전망 (CAGR 31~43%). 한국 에듀테크 시장은 2024년 약 8.7조 원, 2030년 약 13.8조 원 전망 (CAGR 9%) (docs/background/07 §B-2, research-supplement.md §4).

### O2. 교사 행정 부담의 사회적 이슈화

2023년 서이초 사건 이후 교사 번아웃·교권 침해가 사회적 이슈로 부상했다. 교직 만족도 22.7%, 우울 증상 63.2% (doc 01). 교사의 업무 부담을 줄이는 도구에 대한 사회적 공감대가 형성되어 있다.

### O3. 정부 정책 순풍 — AI 디지털 교실 인프라 투자

- 2025년부터 AI 디지털 교과서 도입 (영어, 수학, 정보 교과)
- 6,000개 학교 모니터링 시스템 구축, USD 43.2M 투자 (research-supplement.md §4)
- 15만 명 교원 AI 연수 완료, 1만 명 선도교원 양성 (doc 07 §B-6)
- 디지털활용선도학교 2026년 확대 운영

**시사점:** 교사의 AI 도구 수용성이 급증하고, 교실 내 디바이스 보급률이 높아지면서 [프로젝트명]의 QR 기반 체크인이 실현 가능한 환경이 빠르게 조성되고 있다.

> 단, AI 디지털교과서가 4개월 만에 사실상 폐기된 교훈도 기억해야 한다 (research-supplement.md §3). [프로젝트명]는 "의무화"가 아닌 "자발적 채택" 모델을 취한다.

### O4. 교사 AI 채택 급증 — 주당 5.9시간 절약 효과 입증

Gallup-Walton(2025) 대규모 조사: 미국 공립 K-12 교사의 32%가 주 1회 이상 AI 사용, 주당 평균 5.9시간 절약 (연간 약 6주). AI 도구 사용 교사의 60~84%가 시간 절약을 체감 (research-supplement.md §2-3).

Magic School AI는 2025년 기준 **600만+ 교육자**가 사용 중이며, 교사 주당 10시간 절약 효과를 보고 (research-supplement.md §2-1). 교사가 AI 도구를 적극 채택하는 시장 환경이 이미 형성되었다.

## 9.6 Threats (위협)

### T1. Magic School AI 등 유사 도구의 급성장

Magic School AI는 80개+ 교사 전용 도구를 제공하며 600만+ 교육자를 확보했다. 교사 생산성 도구 시장에서 이미 PMF를 달성한 경쟁자가 존재한다 (research-supplement.md §2-1).

**대응:** Magic School AI는 **수업 전** 준비 도구에 집중하며, Kahoot은 **수업 중** 참여 도구이지만 AI 분석이 없다. [프로젝트명]는 "Kahoot의 게이미피케이션 + AI 인사이트"를 결합한 독자적 포지셔닝을 점유한다. 기존 도구와 경쟁이 아닌 보완 관계로 포지셔닝 가능.

### T2. "LLM 래핑" 비판

AI 교육 도구에 대한 "ChatGPT를 감싼 것 아니냐"는 비판이 존재한다. doc 03 수상작 분석에서도 "ChatGPT 단순 래핑"이 실패 프로젝트의 대표 패턴으로 확인됨.

**대응:** [프로젝트명]의 AI 활용은 단순 텍스트 생성이 아니라 (1) 게이미피케이션 참여 데이터 분석, (2) 교육 맥락(과목·학년·단원) 결합 인사이트 생성, (3) 다음 수업 개선 제안 자동 생성까지 end-to-end 파이프라인이다. 발표에서 시스템 프롬프트 설계 + 도메인 특화 로직을 명시하여 차별화.

### T3. AI 디지털교과서 실패 여파 — 교사의 AI 피로감

2025년 한국 AI 디지털교과서(AIDT) 76종이 4개월 만에 사실상 폐기됨 (research-supplement.md §3). 교사들 사이에 "또 다른 AI 도구?"라는 피로감이 존재할 수 있다.

**대응:** [프로젝트명]는 AIDT와 정반대 접근: (1) 강제 도입이 아닌 교사 자발적 선택, (2) 게이미피케이션으로 학생이 즉시 참여하고 교사도 즉시 가치 체감, (3) AI 결과물 검수 부담 없는 설계 (수치 중심, 낮은 위험도 출력).

### T4. 학교 의사결정 속도의 구조적 느림

에듀테크 B2B 영업 사이클은 5~10년으로 매우 느리다 (doc 09 §3.4). 학교 예산 확보와 도입 결정까지 시간이 오래 걸린다.

**대응:** 초기에는 교사 개인 무료 사용 → 입소문 → 학교/학원 단위 유료 전환(Bottom-up 채택). B2G 의존을 피하고 B2B(학원) + B2C(교사 개인) 병행.

## 10.2 블록별 상세 설명

### Problem (문제)

1. **교사의 가르침이 닿았는지 알 수 없다**: 교직 만족도 22.7%, 우울 증상 63.2% (교사노동조합연맹 2024). 수업 후 학생 이해도를 파악할 체계적 방법이 없다.

### Unfair Advantage (경쟁 우위)

1. **11건 리서치 기반 설계**: 교육 페인포인트 → 경쟁 분석 → 성공/실패 사례 → 학술 근거 → 비판적 재검토를 거친 체계적 기획. 대부분의 해커톤 팀이 갖추지 못하는 깊이.

3. **"피드백 수집 → AI 이해도 분석 → 수업 개선" end-to-end 루프**: 게이미피케이션 참여 도구(Kahoot), 수업 준비 도구(Magic School AI), 학습 관리 도구(클래스팅)는 각각 존재하지만, 피드백 수집부터 다음 수업 초안까지 하나의 루프로 연결하는 도구는 없다 (doc 07 §C-3).

## 11.2 마케팅 전략

### GTM (Go-to-Market) 전략: Bottom-up PLG

**핵심 원칙:** Magic School AI와 ClassDojo가 증명한 PLG(Product-Led Growth) 모델을 따른다. 교사가 제품을 직접 사용하고, 가치를 체감한 후, 동료 교사에게 자발적으로 전파한다.

**Phase 1: 교사 커뮤니티 시딩 (0~3개월)**

| 채널 | 전략 | 예상 효과 |
|------|------|-----------|
| **인디스쿨** | 수학·영어 교사 게시판에 "수업 피드백 자동화 도구 무료 사용기" 작성 | 회원 143,000명+ (전국 초등교사의 약 79%), 설립 2000년 — 국내 최대 초등교사 커뮤니티, 바이럴 효과 최고 (출처: research-supplement.md §6) |
| **참쌤스쿨** | 교사 인플루언서와 협업하여 사용 후기 콘텐츠 제작 | 초등 교사 타겟, 영상 콘텐츠 기반 확산 |
| **교사 카페/밴드** | 과목별 교사 모임(수학교사모임, 영어교육연구회 등)에 무료 체험 초대 | 특정 과목 교사 집단 집중 공략 |
| **에듀테크 해커톤/컨퍼런스** | 발표·부스 운영으로 얼리어답터 교사 확보 | 혁신 성향 교사 직접 접촉 |

**Phase 2: 바이럴 루프 설계 (3~6개월)**

```
교사 A가 [프로젝트명]으로 퀴즈 게임 실행
  → 학생들이 열중하며 참여 (리더보드 경쟁)
  → 수업 후 AI 인사이트 확인 + 다음 수업 개선 제안 자동 생성
  → 다음 수업: "지난 시간 퀴즈에서 많이 틀린 부분을 보강합니다"
  → 학생: "와, 퀴즈 데이터를 보고 수업을 바꿨네" → 다음 게임도 기대
  → 동료 교사 B: "학생들이 저렇게 열중하는 게 뭐야?" → 교사 A가 [프로젝트명] 소개
  → 교사 B 가입 → 루프 반복
```

**바이럴 계수 강화 요소:**
- 퀴즈 게임 중 학생들이 열중하는 모습 자체가 바이럴 — 동료 교사가 궁금해함
- "동료 교사 초대" 시 프리미엄 기능 7일 무료 체험
- 학교 단위 3명 이상 사용 시 자동으로 운영자 대시보드 활성화

**Phase 3: B2B 전환 (6~12개월)**

| 전환 경로 | 전략 |
|-----------|------|
| **학원 → 기관 구독** | 강사 3명+ 사용 학원에 기관 대시보드 무료 제공 → 원장의 가치 체감 → 유료 전환 |
| **학교 → 교육청 파일럿** | 디지털활용선도학교 교사 사용 → 교육청 단위 파일럿 제안 |
| **교사 연수 연계** | AI 교원 연수 프로그램에 [프로젝트명] 체험 세션 포함 제안 |
