# docs/whitepaper/

## 목적
AI 교육 솔루션 프로젝트 기획서 (KIT 바이브코딩 공모전 제출용) 작성 디렉토리.
프로젝트명과 브랜딩은 이슈 #17에서 확정 예정 (현재 `[프로젝트명]` 플레이스홀더 사용).

## 핵심 파일
- **project-plan.md** — 최종 통합 기획서 (ch0~ch12, ~1,737줄)
  - ch0: Executive Summary
  - ch1~ch5: 문제 정의 (프로젝트 개요, 페인포인트, 타겟 사용자, 경쟁 환경, 시장 분석)
  - ch6~ch8: 솔루션 (가치 제안, MVP 기능 정의, PRD)
  - ch9~ch12: 전략 (SWOT, Lean Canvas, GTM 전략, 참고 문헌)

## 핵심 루프 (Core)
학생 피드백 수집 → AI 이해도 분석(%) → 교사 대시보드 → 다음 수업 초안 자동 생성

## UX 레이어
게이미피케이션 (피드백 수집 단계를 Kahoot 스타일로 재미있게 만드는 UX 전략)

## 부가 기능
따봉 업/다운 피드백 (프로젝트 지속 개선 루프)

## 내러티브 원칙
- 핵심 페인포인트: "교사-학생 피드백 루프의 단절"
- Primary Persona: 김서현 (31세, 5년차 수학교사)
- 게이미피케이션은 독립 기능이 아닌 UX 레이어임에 주의
- 프로젝트명/태그라인은 `[프로젝트명]` / `[태그라인 TBD]`로 플레이스홀더 처리

- 프로젝트명 확정 전까지 `[프로젝트명]` 플레이스홀더 유지
