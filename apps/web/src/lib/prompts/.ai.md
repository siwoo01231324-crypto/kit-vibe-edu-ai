# lib/prompts

## 목적
Claude API에 전달할 system/user 메시지 쌍을 생성하는 빌더 함수와 응답 파싱 함수를 모아둔다.

## 파일 구조
- `insights.ts` — buildInsightsPrompt() + InsightSchema(Zod) + parseInsightResponse() — 세션 인사이트 분석
- `class-draft.ts` — buildDraftPrompt() — 수업 초안 마크다운 생성
- `draft-questions.ts` — buildDraftQuestionsPrompt() + DRAFT_QUESTIONS_TOOL(tool_use schema) — 다음 세션 문항 생성
- `session-parse.ts` — buildSessionParsePrompt() + SessionParseSchema(Zod) + parseSessionParseResponse() — 자연어 → 세션 메타데이터 파싱 (#69)

## 공통 패턴
- 빌더 함수는 `{ system: string, user: string }` 반환
- 파서 함수는 Claude 응답(raw string)을 받아 markdown fence 제거 → JSON.parse → Zod 검증 후 타입 결과 반환
- 파싱 실패 시 throw (API route에서 PARSE_ERROR로 래핑)
- MOCK_CLAUDE=1 환경에서는 `callClaude()`가 system 키워드 감지로 고정 응답 반환
