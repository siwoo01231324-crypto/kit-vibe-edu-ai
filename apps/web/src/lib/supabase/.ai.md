# apps/web/src/lib/supabase

## 목적
Supabase JS SDK 클라이언트 래퍼 — 브라우저/서버/관리자 클라이언트를 환경별로 분리 제공.

## 구조
- `client.ts` — 브라우저용 클라이언트 (`@supabase/ssr`의 `createBrowserClient`)
- `server.ts` — RSC/Route Handler용 서버 클라이언트(`createClient`) + RLS 우회 관리자 클라이언트(`createAdminClient`)

## 규칙
- `SUPABASE_SERVICE_ROLE_KEY`는 서버 전용 환경변수. `NEXT_PUBLIC_` prefix 절대 금지.
- `client.ts`는 브라우저 번들에 포함되어도 무방 (anon key만 사용).
- 권한 모델은 RLS 의존 — API 레벨 별도 권한 체크 없이 DB 정책으로 제어.
- ORM(Drizzle, Prisma 등) 금지. Supabase JS SDK 직접 사용.

## 주의
- `createAdminClient`는 RLS를 완전히 우회하므로 **반드시 서버 전용 경로**(`server.ts`, Route Handler, Server Action)에서만 import/사용.
- `client.ts`에서 `createAdminClient` 또는 `SUPABASE_SERVICE_ROLE_KEY` 참조 절대 금지.
- `server.ts`의 `createClient`는 Next.js 15 `async cookies()` 패턴 사용 — `await cookies()` 필수.
