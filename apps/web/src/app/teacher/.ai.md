## 목적
교사 인증이 필요한 모든 페이지를 묶는 Next.js route group.
URL에서 (teacher)는 제거되어 /teacher/dashboard 형태로 접근됨.

## 구조
- `layout.tsx` — 서버 컴포넌트, getUser() 재확인(defense-in-depth), 헤더+로그아웃
- `SignOutButton.tsx` — 클라이언트 컴포넌트, signOut() 후 / 리다이렉트
- `dashboard/` — 교사 대시보드 (스텁, 후속 이슈에서 구현)

## 인증 흐름
1. middleware.ts 가 /teacher/* 경로를 1차 가드
2. layout.tsx 가 getUser() 로 2차 확인 (middleware 우회 방지)
3. 미인증 → redirect('/login')
