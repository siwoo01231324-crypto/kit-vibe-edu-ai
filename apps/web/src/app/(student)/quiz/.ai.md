# (student)/quiz — 학생 퀴즈 응답 페이지

## 목적
학생이 퀴즈 문항에 응답하는 화면. 교사가 세션을 시작하면 waiting 페이지에서 이 경로로 자동 이동된다.

## 라우트
- `/quiz/[sessionId]` → `[sessionId]/page.tsx`
- URL에 `(student)` 그룹명은 노출되지 않음

## 데이터 흐름
```
sessionStorage('studentSession')
  → nickname, sessionId 확인 (없으면 /join 리다이렉트)
  → useStudentQuestions(sessionId)
      - 초기 fetch: questions ORDER BY question_order ASC
      - Realtime: session:{sessionId}:questions 채널 INSERT/UPDATE 구독
  → currentQuestion = questions.find(q => !answeredQuestionIds.has(q.id))
  → handleSelect(choiceIndex)
      - 중복 가드: answeredQuestionIds Set 즉시 추가
      - isCorrect = choiceIndex === currentQuestion.correct_answer (로컬 판단)
      - score = calculateScore(isCorrect, responseTimeMs)
      - supabase.from('responses').insert(...) — anon, INSERT 전용
  → useSessionStatus(sessionId): ended 감지 → 종료 화면
```

## Realtime 채널
- 채널명: `session:{sessionId}:questions` (dev-spec §4.3)
- 이벤트: postgres_changes INSERT, UPDATE on questions

## RLS 제약
- `responses`: anon INSERT 전용. SELECT 불가 → 정답/점수 전부 로컬 계산 후 INSERT 바디에 포함
- `questions`: active 세션에서 anon SELECT 허용

## 주요 파일
| 파일 | 역할 |
|---|---|
| `[sessionId]/page.tsx` | 퀴즈 페이지 (클라이언트 컴포넌트) |
| `../../hooks/useStudentQuestions.ts` | questions 초기 fetch + Realtime 구독 훅 |
| `../../lib/scoring.ts` | calculateScore(isCorrect, responseTimeMs) |
| `../../hooks/useSessionStatus.ts` | 세션 종료(ended) 감지 |
